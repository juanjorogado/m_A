# Plan: Marcus Aurelius Meditations PWA

## Objective
Convert the existing React web prototype to a Progressive Web App (PWA) that can be installed and used on iPhone via Safari without requiring Apple Developer Program payment or TestFlight.

## Current State
- Web-based React component showing daily Marcus Aurelius meditation
- Card flip animation on click/tap
- Styling using Tailwind-like CSS classes
- Hardcoded data for 2026-05-25

## Target Platform
Progressive Web App (PWA) using standard web technologies (React) optimized for iOS Safari installation and offline use

## Implementation Plan

### 1. Project Setup (remains web)
- The existing code is already a React web app; we'll enhance it to be a PWA.
- Ensure we have a recent version of React and possibly use Create React App or Vite if needed.
- Install required dependencies for PWA: 
  - `workbox-webpack-plugin` or `vite-plugin-pwa` depending on setup.
  - Or we can manually add service worker and manifest.

### 2. PWA Specifics
- Add a web app manifest (`manifest.json`) with icons, name, short_name, start_url, display: "standalone" or "fullscreen", background_color, theme_color.
- Add a service worker to cache assets and enable offline functionality.
- Ensure the app is served over HTTPS (required for service workers on iOS Safari, except localhost).

### 3. Adaptations for iOS
- iOS Safari has specific requirements for PWAs to be installable:
  - Manifest must include proper icons (various sizes).
  - Use `apple-mobile-web-app-capable`, `apple-mobile-web-app-status-bar-style`, `apple-mobile-web-app-title` meta tags.
  - Provide a startup image if needed.
- The service worker must be registered and handle fetch events to cache the app shell and content.

### 4. Component Adjustments (minimal)
- The existing React components can remain largely the same.
- Ensure that the UI is responsive and works well on mobile screens.
- Consider adding a meta viewport tag if not present: `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
- The card flip animation should work via CSS as before.

### 5. Offline Functionality
- Cache the main HTML, CSS, JS, and possibly the meditation data (if we want it to work completely offline, we can store the data in the service worker cache or use localStorage).
- Since the data is small and hardcoded, we can include it in the initial load and cache it.

### 6. Testing Strategy
- Test locally by running the web app and using Lighthouse to audit PWA compatibility.
- On iPhone, open Safari, navigate to the app's URL (served via localhost or a public HTTPS URL), then use the "Share" button and select "Add to Home Screen".
- Once installed, the app should launch without Safari UI and work offline.

### 7. Distribution Options
- Host the PWA on any static web host (GitHub Pages, Netlify, Vercel, etc.) with HTTPS.
- Users can install it directly from Safari without going through the App Store.
- No Apple Developer Program payment required.

### 10. Next Steps
1. Set up or verify the React project setup (if not already using a bundler that supports PWAs).
2. Add the web app manifest and configure meta tags for iOS.
3. Implement a service worker to cache assets and enable offline use.
4. Test the PWA locally and on iPhone via Safari.
5. Deploy to a static HTTPS host for easy sharing.

### 11. Success Criteria
- The app can be visited in Safari on iPhone.
- The user can add it to the Home Screen via the Share button.
- When launched from the Home Screen, it opens without Safari UI (standalone).
- The core functionality (daily meditation, card flip) works online and offline.
- No payment to Apple required for distribution or installation.

### 12. Notes
- PWAs on iOS have some limitations compared to native apps (e.g., no push notifications, limited background access), but for this simple app, they are sufficient.
- The first time the app is launched, it will need to load from the network; subsequent launches can be faster due to caching.
- We must ensure the server serves the service worker correctly (with proper MIME types).
